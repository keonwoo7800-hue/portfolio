▸ 문제 상황
약 40중화 규모의 대형 운영 인프라 환경에서 간헐적으로 약 10분간 지속되는 서비스 장애 및 사용자 세션 유실 현상 발생
장애 시점의 WebtoB 및 JEUS 시스템 로그 교차 분석 → 간헐적 네트워크 절단 이슈 확인
JEUS 세션 서버 로그 추적 → 네트워크 일시 단절 발생 시 솔루션의 failover-delay 타임아웃 이내에 인프라 복구가 완료되지 못해 세션 유실 발생
▸ 개선 접근 방식
[OS 레벨 커널 튜닝] 인프라 장애 발생 시 OS 레벨에서 신속히 네트워크 단절을 인지하고 복구 프로세스를 가동하도록 TCP Parameter(Keepalive) 값 최적화 조정
[솔루션 레벨 Health-Check 최적화] WebtoB와 JEUS 간 실시간 상태 체크 동기화를 긴밀히 유지하기 위해 WebtoB 및 JEUS의 SvrChkTime 및 JEUS Read-Timeout 파라미터 유기적 튜닝
[세션 보존 가용성] 순간적인 인프라 끊김이 발생하더라도 세션이 유실되지 않도록 failover-delay 수치 조정
▸ 개선 결과
대규모 다중화 환경에서 네트워크 이슈 발생 시, 기존 10분 이상 소요되던 복구 목표 시간(RTO) 및 서비스 다운타임을 1분 미만으로 단축
개선 조치 적용 후 수행된 가용성 테스트 과정에서 세션 유실 현상 0건 달성
