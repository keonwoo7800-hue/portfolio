▸ 문제 원인 분석

▸ 해결 접근 방식 — 구조적 개선
JS·JSON 등 텍스트 기반 Static Resource에 Gzip Compression 전송 적용
반복 호출 대상 리소스에 Cache-Control 헤더 및 캐싱 옵션 최적화
기존 WAS 단에서 수행하던 대용량 JS 번들 파일의 I/O 및 전송 처리를 WebtoB의 내부 캐싱 및 파일 처리 엔진으로 이관하여 WAS 부하 근본적 차단
긴급 부하 테스트 수행 → Worker Process 수 및 Timeout 파라미터 재튜닝
▸ 결과 및 인프라 변화
Netfunnel Active Session 최대 허용 수 : 1,500 → 10,000 유저 이상 (약 6.7배 향상)
명절 대수송 기간 서비스 지연 건수 및 네트워크 대역폭 사용량 대폭 감소

▸ 문제 원인 분석
40중화 대규모 환경에서 간헐적으로 약 10분간 서비스 장애 및 세션 유실 반복 발생
장애 발생 시간대 WebtoB·JEUS 시스템 로그 분석 → 네트워크 절단(Network Disconnection) 이슈 의심
세션 로그 분석 → Failover-Delay 구간 내 장애 미복구로 인한 세션 유실 확인

▸ 해결 접근 방식
OS 레벨 : TCP Keepalive 파라미터 조정 → 네트워크 절단 조기 감지 및 빠른 복구 대응
솔루션 레벨 : WebtoB SvrChkTime·Read-Timeout 값 재조정 → 연결 이상 감지 임계값 최적화
애플리케이션 레벨 : JEUS Failover-Delay 수치 조정 → 동일 장애 발생 시 세션 보존 가능성 향상
OS·솔루션·애플리케이션 3개 레이어 연계 파라미터 최적화를 통해 연쇄 장애 확산 차단 및 인프라 고가용성(HA) 확보
▸ 결과 및 인프라 변화
서비스 장애 지속 시간 : 10분 이상 → 1분 미만으로 단축 (90% 이상 개선)
이후 가용성 테스트(HA Test) 수행 중 세션 유실 발생 없음 확인
TCP·솔루션·애플리케이션 파라미터 기준값 정립 → 유사 대규모 환경 적용 시 재활용 자산으로 공유
